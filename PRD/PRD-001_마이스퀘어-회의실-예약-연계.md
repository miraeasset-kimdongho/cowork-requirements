# PRD-001: 마이스퀘어 연계 회의실 예약

> **제품**: Mattermost Calendar 플러그인
> **작성자**: 권현준 매니저
> **작성일**: 2026-06-22
> **최종수정**: 2026-06-22
> **상태**: `Draft`
> **관련 이슈**: -
> **RFQ 영역**: 통합/연계 (M)

## 0. 전제 & 제약

- 회의실(자원) 예약/조회/수정/삭제는 **기존 마이스퀘어 자원예약 API를 그대로 호출**한다. 신규 예약 백엔드는 만들지 않는다.
- 신규 DB는 **두 가지만**: `outlook_calendar`에 예약 연동 컬럼 4종 추가 + `user_favorite_room` 테이블 신설(§2).
- **회의실 함께보기 목록은 프론트에서 저장**한다(사람 함께보기와 동일 방식, DB 미사용).

## 1. 개요 & 범위

캘린더 일정 등록 흐름 안에서 회의실을 선택·예약하고, 캘린더에서 회의실 일정을 함께 조망하는 기능. 예약/조회/수정/삭제는 기존 자원예약 API로 처리한다.

### 포함(In-Scope)
- **DB**: `outlook_calendar` 예약 연동 컬럼 + `user_favorite_room` 신설
- **회의실 함께보기**: 캘린더에 '회의실 함께보기' 줄/추가 버튼, 추가 팝업(트리·체크박스), 회의실 일정 표출(7/3/1일 뷰에서)
- **일정 등록**: 장소 아래 '자원 예약' 체크 → 관심회의실·시간표 인라인 → 예약 폼 → `자원 예약하기`/`일정 등록하기`
- **일정 수정**: 시간 변경 시 자원예약 동반 수정
- **일정 삭제**: 연결 예약 동반 삭제 확인
- **일정 상세**: 장소 표출 정책
- 데스크탑 + 모바일 각 화면

## 2. 데이터 모델

### 2.1 `outlook_calendar` (기존 아웃룩 일정 테이블) — 신규 컬럼 4종 추가

| 신규 컬럼 | 타입 | 설명 | API 매핑 |
|---|---|---|---|
| `date_id` | varchar(20) | 예약 **회차(occurrence)** ID | API `DateID` (§8.2) |
| `booking_id` | varchar(100) | 자원예약 **묶음** 번호 ID | API `BookingID` (§8.2) |
| `booking_start_date` | timestamp | **예약 시작시간** (일정 시간과 별개) | 예약등록 body |
| `booking_end_date` | timestamp | **예약 종료시간** | 예약등록 body |

- 예약 연결이 없으면 4개 컬럼 모두 `NULL`.
- 일정 시간(`start_date`/`end_date`)과 예약 시간(`booking_start_date`/`booking_end_date`)은 **독립**(예약 시간 ≠ 일정 시간 가능).
- `date_id`가 자원 예약의 주요 키. (booking_id는 안 쓸 확률이 높음 -> 반복 등록 지원을 안하기에)

### 2.2 `user_favorite_room` (신규 테이블) — 관심회의실
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `id` | varchar(20) | PK |
| `user_id` | varchar(255) | 사용자 ID |
| `resource_id` | varchar(50) | 회의실(자원) ID = API `ResourceID` |
| `sort_order` | integer | 정렬/표시 순서 |
| `created_at` / `updated_at` | timestamp | 생성/수정 |

- 일정 등록 시 관심회의실로 선택·저장한 회의실 보관(§4.2).

### 2.3 저장 정책 요약
- **관심회의실** → `user_favorite_room` (DB)
- **회의실 함께보기 목록** → **프론트 저장**(DB 미사용)
- **예약 상세/현황** → **마이스퀘어 예약현황 API 라이브**(별도 예약 테이블 미생성)

## 3. 회의실 함께보기 (기본 캘린더)

일정 등록과 무관한 **캘린더 첫 화면** 기능. 기존 '함께보기'(사람) 아래에 '회의실 함께보기'(회의실) 줄을 추가해, 회의실 일정을 사람 일정과 함께 조망한다.

### 3.1 진입 — '함께보기' / '회의실 함께보기' 2줄 고정
- 캘린더 헤더 아래에 **두 줄을 고정** 표시:
  1. `함께보기` : 등록된 사용자 칩 + 우측 `[+ 함께보기]`
  2. `회의실 함께보기` : 등록된 회의실 칩 + 우측 `[+ 회의실 추가]`
- 두 버튼으로 **언제든** 사용자/회의실을 추가할 수 있다.
- (참조: v4 PPT — 회의실 함께보기 ①)
- 단 월간 뷰에서는 회의실 함께보기는 제공하지않는다 ( -> 예약현황 API가 과하게 길 수도 있음)

### 3.2 회의실 추가 팝업 — 트리 + 체크박스
- `[+ 회의실 추가]` 클릭 시 팝업 오픈.
- **관심회의실 선택과 동일한 방식**: 자원목록 API로 **트리 구조**(ROOT/Folder/RESOURCE) 표출, **체크박스**로 다중 선택.
- 선택·저장한 회의실은 팝업 하단에 **칩으로 표시**(✕로 제거) — 사용자 추가 팝업과 동일 패턴.
- 목록은 **프론트 저장**(DB 테이블 없음).

### 3.3 회의실 일정 표출
- 등록한 회의실의 일정이 **사람 함께보기처럼 같은 캘린더 뷰에 열로 공존**.
- **뷰 제약**: 회의실 일정은 **7일 / 3일 / 1일 뷰에서만** 표시.
  - **월(Month) 뷰**: 회의실 일정만 미표시. **사람 함께보기는 월 뷰에서도 표시**(월 뷰 자체는 정상 동작).
- 마이스퀘어에서 사용자가 해당 회의실을 관심회의실로 등록하지 않았다면 **데이터 없음** → 빈 열로 그대로 표출.

## 4. 일정 등록

일정 등록 팝업 하나에서 처리하되, **동작 버튼을 분리**한다: `[자원 예약하기]`(예약등록) · `[일정 등록하기]`(일정 저장).

### 4.1 진입 — 장소 아래 '자원 예약' 체크
- 일정 등록 팝업의 **장소 필드를 맨 아래**로 두고, 그 아래에 **`자원을 예약하시겠습니까?` 체크박스** 추가.
- 체크 시 팝업 **아래로 이어서**(같은 팝업, 스크롤) 관심회의실·회의실 시간표·예약 폼이 인라인으로 펼쳐진다(새 팝업 아님).

### 4.2 관심회의실 (자원목록 API 트리)
- **자원목록 API**로 트리 구조 표출, 체크박스로 선택.
- **선택 순서대로** 회의실 시간표에 열이 추가된다(①②③…).
- 선택 결과는 **`user_favorite_room`에 저장**한다.
- **재방문 규칙**: 한 번이라도 저장한 사용자는 다음 일정 등록 시 트리를 펼치지 않고 **최상단 `＋ 관심회의실 등록하기` 줄만** 노출(필요 시 펼쳐 추가).

### 4.3 회의실 시간표
- 관심회의실로 선택한 회의실들의 시간표를 표출.
- **기본 날짜** = 위 일정에서 선택한 날짜(과거 불가). 위에서 선택하지 않았으면 **오늘**.
- **시간대 클릭** → 아래 **예약 폼**을 노출하고, **시간·장소(회의실)를 기본 매핑**한 뒤 **포커스 이동**.

### 4.4 예약 폼 & 버튼 흐름
- 예약 폼 필드(§9): 예약자·용도·예약기간(시작/종료)·자원명·자원분류·반납/예약절차·최소대여 등(기존 예약신청 항목 그대로).
- 정보 입력이 끝나면 **`[자원 예약하기]`** 클릭 → **예약등록 API** 호출.
- **결과는 버튼 바로 아래 빨간 줄**로 실시간 표시(성공/실패).
- 그 아래 **`[일정 등록하기]`** 버튼으로 일정을 저장(성공 시 `booking_id`/`date_id`/예약 시간 함께 기록).
- 기존 예약신청 포맷은 추후 추가 예정

### 4.5 실패 / 예외 알림
- **`[자원 예약하기]` 실패 후 `[일정 등록하기]`** 를 누르면:
  > **"자원예약이 실패하였습니다. 일정만 단독 등록하시겠습니까?"** → `예`(일정만 등록, 예약 정보 폐기) / `아니오`(예약 재시도)

---

## 5. 일정 수정

- **일정 시간이 수정될 경우**, `자원예약도 같이 수정하시겠습니까?` **체크박스가 등장**.
- 체크 시: 일정에 저장해 둔 **예약번호(`date_id`=DateID)** 로 **예약현황에서 기존 예약을 찾아 세팅**한 뒤 수정한다.
- 이후 흐름은 등록과 동일: `[자원 예약하기]` → 빨간 줄 결과 → `[일정 등록하기]` → 실패 시 §4.5 알림.
- 체크하지 않으면 일정만 수정(예약 변경 없음).

---

## 6. 일정 삭제

- 삭제 시 일정의 **자원예약ID(`date_id`)가 NULL이 아니면** 확인 알림:
  > **"자원예약도 같이 삭제하시겠습니까?"** → `예`(예약도 삭제) / `아니오`(일정만 삭제, 외부 예약은 유지)
- 예약 삭제 호출은 `date_id` 기준(§8).

---

## 7. 일정 상세보기

- 상세보기에는 **일정 장소(location)만** 표출한다.
- 회의실(자원) 장소·예약 상세 등은 **표출하지 않는다**(단순 표시).
