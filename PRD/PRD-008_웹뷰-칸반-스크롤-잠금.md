# PRD-008: View 모드 기능 추가 (웹뷰 편의성 향상)

> **제품**: Mattermost 웹뷰
> **작성자**: 김동호 매니저
> **작성일**: 2026-06-22
> **최종수정**: 2026-06-25
> **상태**: `Draft`
> **관련 이슈**: -
> **RFQ 영역**: 부가기능 (M)

---

## 1. 한 줄 요약 (One-liner)

> Mattermost 웹뷰로 Jira/Confluence 접근 시, 스크롤·확대/축소 등 View 컨트롤 외 조작을 막는다.

---

## 2. 배경 & 문제 (Why)

- 현재 상황 (As-Is): 웹뷰에서 보드 조작 중 의도치 않은 이벤트(드래그 등) 발생 가능
- 발생하는 문제/불편: 조회 목적인데 실수로 보드가 변경될 위험 및 가시성 저하
- 왜 지금 해야 하는가: 웹뷰 조회 안정성 확보

---

## 3. 핵심 목표 — 딱 하나 (Focus)

- 웹뷰에서 **뷰 조작만 허용하고 그 외 조작을 잠근다.**

---

## 4. 하지 않을 것 (Non-Goals)

- 데스크탑 네이티브 Jira 보드 동작 변경
- 칸반 보드 자체 기능 수정

---

## 5. 성공 기준 (Measurable Success)

| 지표 | 현재 | 목표 | 측정 방법 |
|------|------|------|-----------|
| 웹뷰 내 의도치 않은 보드 변경 | 발생 가능 | 차단 | QA |
| 스크롤 조회 정상 동작 | - | 유지 | QA |
| 줌인/줌아웃 허용 | 불가 | 허용 | QA |

---

## 6. 사용자 & 시나리오

- 대상 사용자: Mattermost 웹뷰로 Jira/컨플을 보는 사용자
- 사용자 스토리:

```
As a 웹뷰 사용자,
I want to 컨플/지라를 뷰 조작(스크롤·줌)만으로 보고,
So that 실수로 카드가 이동/변경되지 않으면서 편하게 확대·축소하며 확인한다.
```

## 8. 제약 / 가정 / 의존성

- **제약**: 웹뷰 컴포넌트의 이벤트 제어 가능 범위
- **가정**: 칸반 보드 DOM/이벤트 식별 가능
- **의존성**: Mattermost 웹뷰

---

## 9. 디자인 / UI

- 기존 메터모스트 웹뷰라 별도 목업은 필요 없음.
- 주요 화면 흐름: 웹뷰로 Jira/컨플 진입 → View 모드(스크롤·줌만 허용) 적용
- `< >` (메터모스트 내 history 컴포넌트를 조작하는 기존 버튼)의 UI가 강화되길 희망
